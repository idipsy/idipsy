▶本次改动说明（VIP列表页 / VIP晋升机制调整）— 点击展开查看变更详情

一、字段变更

  • 移除：有效投注流水（原晋升判定依据，不再使用）
  • 新增：抽佣系数NEW — 按等级独立配置，必须大于0；真实判定用，不对客户端展示
  • 新增：晋升所需经验值门槛（增量）NEW — 与原「有效投注流水」的逐级累加口径保持一致：这里填的是在上一级基础上新增的门槛值，不是绝对总量。举例：VIP1填100、VIP2填200、VIP3填300，会员要升到VIP3，实际需要的累计经验值 = 100+200+300 = 600，而不是直接看VIP3这一行填的300。系统会自动计算"累计门槛"这一只读列，方便核对
  • 晋级礼金 / 周流水 / 每周福利礼金 / 90天保级流水：字段不变，统一补充"U"金额单位
  • 90天保级流水、发放礼金稽核倍数、发放方式：字段与逻辑均不变

二、晋升判定算法：累积经验值 + 阶梯式跨级（原逻辑为流水累计，本次替换）

  • 每日03:00，将会员T-1日预期抽佣金额 × 当前等级的抽佣系数，计入会员持久化的经验值余额，每天算出来的增量会持续累加进同一个余额里，跨天保留，今天没达标的部分不会清零，会一直留到累加够门槛为止。会员的预期抽佣金额数据取自【报表-会员盈亏-按会员-预期抽佣金额】
  • 从当前等级+1开始逐级比对：用累加后的经验值余额，和"从VIP1到目标等级的累计门槛"比较（累计门槛=各级增量门槛之和，与原「有效投注流水」逻辑一致）。达标则继续判断下一级（支持同日跨越多级），一旦某级不达标即停止（阶梯式，不能跳过未达标等级触达更高等级）
  • 达标晋升后，超出门槛的余额无缝结转，继续计入下一级门槛判定；未达标部分保留至次日继续累加，不清零、不作废
  • 会员被90天保级机制判定降级时，经验值余额一并清零，重新开始累积（保级判定逻辑本身不变，仍按原有效流水口径独立运行）

三、跨级晋级礼金：依次发放

  • 单日一次性跨越多级时（如VIP1→VIP5），按跨越的每一级依次生成独立记录，写入【VIP福利彩金审核】列表，而非只发放最终等级的礼金
  • 每条记录按该等级自身的"发放方式"独立处理：自动发放 → 直接标记为已审核通过；人工发放 → 进入人工审核队列，不影响同批次其他自动发放等级

四、客户端展示设置：新增功能，设计为"全局统一配置"

  • 具体实现：工具栏新增独立的「客户端展示设置」入口（不放进单个等级的新增/编辑抽屉），勾选结果对所有VIP等级统一生效
  • 同一入口内合并了规则说明文案编辑（VIP等级晋升规则说明 / 保级流水晋升规则说明 / 周流水晋升规则说明），同样全局生效；三段文案均为非必填项，若后台未填写，客户端对应模块不展示
  • 点击工具栏「客户端展示设置」按钮可查看该弹窗的完整效果

五、安全设计：抽佣系数为何不对客户端展示，且采用精确值而非区间化

  • 目的：防止用户通过自己的投注/输赢数据反推真实抽佣比例，即游戏利润空间
  • 核心防线是"客户端不暴露真实经验值增量/余额"，而不是"后台取值精确还是分档"——客户端只展示"是否已达到"的状态，不展示任何具体数值或进度百分比，因此攻击者始终拿不到可用于反推的真实输出，后台维持精确值 × 系数计算即可，无需额外做区间化
